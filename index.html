About Me

I was born and raised in Bangladesh. I have been good at mathematics and have loved solving mathematical problems since childhood.

In 2011, I completed a Bachelor's degree in Mathematics at the University of Chittagong, Bangladesh. In 2012, I graduated from the same university with a Master's degree in Applied Mathematics. During this time, I learned various programming languages, including C and C++. I was awarded three gold medals for excellence in mathematics during my bachelor's and master's studies.

In 2015, I pursued a second Master's degree in Mathematics at the Norwegian University of Science and Technology (NTNU), Norway. Among other courses, I studied Cryptography and used MATLAB during my studies there.

In 2019, I led the teaching of PYTHON for grade 7 when programming was introduced in the Rudolf Steiner School, Gothenburg. I designed, prepared lectures, and taught the course. While teaching the course, I developed a keen interest in computer languages and enjoyed creating beautiful and functional applications.

In 2022, my interest and capability in mathematical problem-solving led me to pursue a formal education in Computer Science with a focus on Web Development at the Blekinge Institute of Technology, aiming to become a certified full-stack developer and launch a career in tech.

I worked as a summer worker at Scania in 2023. In my role with the Energy Domain Data Analytics team at Scania, I utilized Python, SQL, Snowflake, and Streamlit for data retrieval, processing, and visualization through plots and dashboards. Recognizing the critical importance of data lineage in ensuring data quality and reliability, I continued my work with the team for my thesis, focusing on data lineage and ETL processes. In my thesis, I tackled challenges such as tracking changes in the data warehouse, SQL queries, and dashboards. For instance, dashboards referencing non-existent database elements caused SQL queries to fail, disrupting operations. Similarly, upstream changes like removing or modifying column names impacted dashboard queries, and outdated documentation hindered stakeholders from accessing accurate information. Additionally, I am currently enrolled in online courses on machine learning and Grafana to refine my expertise in data science, with a focus on visualizing metrics, analyzing time series data, and enhancing application analytics.

Data Analysis with Python

This individual examination project was part of the Mathematical Modelling course. I used Python, along with libraries such as pandas and scipy, to analyze the air temperature data for three different temperature stations (Arvidsjaur A, Abisko Aut, Adelsö A) in Sweden. When the developed Python script is run, the temperature data is retrieved dynamically using an open data API from the Swedish Meteorological and Hydrological Institute (SMHI). The latest 1000 data samples from each station are retrieved, pre-processed, and converted to Python pandas’ data frame for statistical analysis.

Thesis: Visualizing Data Lineage & Automating Documentation for Data Products

This thesis is submitted to the Faculty of Computing at Blekinge Institute of Technology in partial fulfillment of the requirements for the degree of Higher Education Diploma in Software Engineering with emphasis in Web Programming. This projcet is done in collaboration with Scania CV AB, Sweden.

In this project, a Python backend is developed that parses SQL queries from various dashboard panels and generates an abstract syntax tree (AST) for each query. The ASTs are used to create a JSON file representing the lineage data in a hierarchical structure. This JSON file provides information about which column is used from which table, which table is used from which schema, and which schema is used from which database. The JSON-formatted data is compared with the data warehouse's database structure to identify failed SQL queries in dashboards. In addition, a frontend is developed, primarily using Vue.js and D3.js, to visualize the data lineage, highlight the failed parts of the dashboard queries, and host documentation for the dashboards. Whenever an AST is modified, the corresponding documentation page is automatically updated to reflect the changes.
